3. LLM, 믿고 쓸 수 있을까요? 지표 중심 LLM 모니터링
발표자: 송인서 | 업스테이지 SW엔지니어
시간: 10:20 ~ 11:00
강의 내용:
1) LLM 운영의 어려움
- 고객 요구사항 불명확 → 리소스 요구 파악 어려움
- LLM은 토큰 단위 응답 생성 → 비용·성능 예측 어려움
- 서비스 운영 후 모니터링 기반 개선 필요

2) LLM 추론 과정
- 입력(프롬프트) 처리 후 토큰 생성 시작
- 출력은 토큰 단위로 순차 생성
- 내부 계산 결과 캐싱 가능 (KV 캐시 등)

3) 주요 모니터링 지표
- TTFT (Time To First Token): 첫 토큰 생성 시간, 낮을수록 좋음
- ITL / TPOT: 토큰 생성 지연 시간, 낮을수록 좋음
- TPS (Tokens Per Second): 초당 생성 토큰 수, 높을수록 좋음
- Latency: 전체 요청 처리 시간, 모델/서비스 경향 파악용
- Input/Output Tokens: 요청 워크로드 확인 및 비용 관리
- System Metrics: GPU, 메모리 등 자원 활용 모니터링

4) 운영 팁
- 단일 지표로 성능 판단 불가 → 종합적 해석 필요
- 프롬프트 캐시 활용: 정적 정보 상단 배치
- 서비스 특성에 따라 중요 지표 달라짐
